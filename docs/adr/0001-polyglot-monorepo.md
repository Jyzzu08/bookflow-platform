# ADR-0001: BookFlow Polyglot Monorepo

Date: 2026-03-13

## Context
The legacy project had fragmented services and weak operational standards.

## Decision
Adopt a monorepo with polyglot services, API gateway compatibility layer (v1), and contract-first v2 APIs.

## Consequences
- Better developer experience and CI visibility.
- Higher operational consistency across services.
- Slightly higher initial setup complexity.
