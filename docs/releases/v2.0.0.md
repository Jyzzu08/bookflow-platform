# Release Notes v2.0.0

## Highlights

- CI validation is stricter and now works on clean clones by preparing `.env` before `docker compose config`.
- The React + Vite web app builds cleanly again thanks to proper `import.meta.env` typing.
- Trivy now fails deterministically on `HIGH` and `CRITICAL` findings, matching local and GitHub validation behavior.
- Maintained legacy services were refreshed instead of excluded: Node runtimes moved to 22, Flask moved to 2.3.x, and vulnerable lockfiles were regenerated.
- Local setup documentation now matches CI expectations, reducing drift between development and repository checks.

## Compatibility

- Legacy `v1` routes remain unchanged.
- Modern `v2` contracts remain unchanged.
- This release focuses on hardening, security hygiene, and reproducible validation rather than API-breaking behavior.
